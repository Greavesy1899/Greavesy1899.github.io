The aim for this project was to implement a simple boids algorithm and build a game which had some basic combat mechanics, as well as networking by using an engine called "Urho3D". The algorithm given was extremely slow and could only handle about 100-250 boids. The end result was very positive; the game could use about 2,5000 - 5,000 boids with an average of 30FPS.

When the program is computing forces for boids, the main issue is that the algorithm complexity would be quadratic; the original code would loop through all the boids twice in a single update function. When optimizing the boids, three improvements were made; The first was to implement a "Spatial Grid", which essentialy decreases the amount of boids which are used to calculate forces in nearby cells in the spatial grid. The final optimization was implementing multi-threading; this was a major boost to performance, and the boids were split depending on the amount of threads, as boids may fail to update before the update function is processed on the main thread and cause jittering effects on the boids.

During this assignment, I wanted to make it feel more like a game, so an extra level was added, a main menu, options with a working configuration file and a level select screen. For gameplay, I made a basic turret defense where players can take control of turrets and target the boids in the sky. These were done using Urho3D's componenet system, was transferring data over the network is really easy to do.
